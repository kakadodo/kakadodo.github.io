## 挑戰心得:
這次對我來說比較像是切版的細心度挑戰，設計稿總共有三個版面，雖然區塊中的元件都是重複性的，但元件本身並不簡單啊..光一個元件裡面的層級就有5.6層以上了..更不用說再搭配 hover 產生的額外內容~切一切都會懷疑自己這樣做是對的嗎XD(已迷失方向)

元件中有不少元素排列的位置不一，用 flexbox 輕鬆解決~怎麼會這麼好用啊!!!!如果用一般切法的話，可能就會一堆 float 物件加特定的 padding 跟 margin，以及因為 float 而需要另外追加的 clearfix，flexbox 可以完全解決這類困擾!超推超推~

介面基本上都盡量照著設計稿刻，但圖表這塊就只能"盡力表現"了~之前一段時間接觸 D3.js，這次決定來使用它的延伸產品 C3.js~
C3 官網直接了當的說明函示庫是基於 D3 引擎而來的~要使用 C3 的話，要先載入 D3 的函式庫喔~
之前會想說既然還要載入 D3 幹嘛不直接用 D3 就好，這次實作下來的理解是... C3 在產生圖表過程會更直覺一點~把圖表需要呈現的配置塞在一個物件中，在呼叫 C3 去套用這個自訂義的配置物件，然後~~~~圖表就出來了!!
而且 C3 本身對圖表有基本的 CSS 設定，因此產生出來的圖表其實就很 OK 了!圖表的顏色呈現也可以直接在配置物件中設定~總之!要對圖表做甚麼設定，全部寫出來就對了! `c3.generate({配置內容})` 圖表就會出現在畫面上!
```js
var chart = c3.generate({
    bindto: '#chart', //跟 Vue 一樣，會需要設定綁定的 html 元素
    data: {
      columns: [
        ['data1', 30, 200, 100, 400, 150, 250],
        ['data2', 50, 20, 10, 40, 15, 25]
      ]
    },
    //其他配置設定...
});
```
C3 在處理圖表的動畫似乎挺強大的!看官網的 examples 寫的一副超輕鬆的感覺..如果這些要用 D3 來寫的話..可能會佔不少版面跟時間!雖然 D3 的彈性真的非常大~但就變成需要一步一步地去設定這些參數，變成無限長的`.`連接(跟 jQuery 一樣)，久久一看可能就無法一下看懂自己在寫甚麼..

這次還有一塊覺得很煎熬的是...下拉選單，頁面中的下拉選單超多~~~雖然用 bootstrap 可以很容易地把下拉選單生出來..但有碰到一個小問題，當下拉選單中塞入 checkbox 這種需要於特定位置點擊的元件時，會變成不好觸發..因為點不對位置選單就會當作是取消選取而變隱藏，這可能要再想想有沒有更好的解決方式 :p
